实现一个自己的脚本语言（绪）

Mslxl
  2022-11-01 2022-11-01 创建   2024-07-24 11:49:53 2024-07-24 11:49:53 更新  
  • Compiler 

说实话很久以前我就有这个想法了，不过近几个月才开始动工。先写篇占坑吧，也提醒自己。

语言设计的话打算实现一个静态强类行的脚本语言，运行时先将脚本编译成自己的虚拟机指令再解释执行。目前我想先不把性能放在首位上，而是先实现一些语言特性（比如模式匹配等），也就是写的爽优先。

就我个人倾向来说是不太喜欢用散列表的，所以在这个虚拟机里我想尽可能少用散列表。

目前写成这个样子了 GitHub - mslxl/autoscript: A simple toy language, work in progress ，目前已经能跑一些函数了，但是在实现结构体(OOP)和 GC 的方法上还没有头绪，之后有什么想法的话可能会一起发出来。
